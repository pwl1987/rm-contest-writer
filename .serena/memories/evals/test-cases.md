# rm-contest-writer 技能测试用例

## 测试目标
验证步骤条交互模式的各项功能

## 测试用例列表

### 1. 基础流程测试
- **ID**: basic-flow
- **输入**: 依次选择所有步骤
- **预期**: 生成完整的 contest prompt

### 2. 单选级联测试
- **ID**: cascade-single
- **输入**: 选择 track=商业化, ai_mode=本地部署
- **预期**: 痛点列表更新为"成本"相关选项

### 3. 多选+自定义测试
- **ID**: multi-custom
- **输入**: 选择痛点=[成本问题, 效果不佳], 自定义痛点=其他
- **预期**: 合并为4个痛点项

### 4. 默认值保持测试
- **ID**: default-persist
- **输入**: 只选择track, 其余全部默认
- **预期**: 使用默认模型和模板

## 验证标准
- 步骤导航正确
- 级联更新生效
- 自定义输入被正确合并
- 生成 prompt 包含所有选择
