# Phase 3 Plan: 解决自评虚高问题

## Context
当前 SKILL.md Step 5 评分卡存在 3 个导致自评虚高的设计缺陷：
1. Gate 执行软化：BLOCK 状态仅提示修改建议，未强制停止文档生成
2. 无 Delta 显示：不展示自评分与客观分的差值，缺乏校准反馈
3. 评审者角色弱：prompt 未明确要求对抗性挑刺，易产生自我辩护

目标：使模型自评分从 T:9 回落至 ~T:6.5，与实际输出质量对齐。

## 计划 03-01: Gate BLOCK 强制执行
- 修改 SKILL.md Step 5 标题 + 5 处 Gate 语言
- 标题改为 "终稿质量评分卡（Gate 强制校验 + FAIL即停）"
- 评分卡前添加执行规则强调
- 5 个 Gate 语言统一改为 "立即停止文档生成（不输出正文，不进入第六步）"
- 综合评分 < T:7 时添加 BLOCK 行

## 计划 03-02: 自评-客观 Delta 显示机制
- 在评分卡后新增 5.1 节 "评分汇总与 Delta 报告"
- 添加基准分对照表：公文格式 T:8、内容完整度 T:6、逻辑严谨性 T:5、语言专业性 T:7、技术深度 T:6，综合基准 T:6.5
- 若 Delta > 0，强制重新审视输出

## 计划 03-03: 对抗性评审 Prompt 优化
- Step 5 头部新增反自我辩护规则（4 条强制规则）
- 评审者角色升级为"严苛的甲方评审"
- 禁止自我辩护、必须引用 GB/T 9704-2012 条文编号

## 验证方法
1. grep -n "立即停止文档生成" SKILL.md → 应有 6 处
2. grep -n "Delta" SKILL.md → 应有 1 处
3. grep -n "禁止自我辩护" SKILL.md → 应有 1 处
4. 模拟测试：低质量草稿触发 BLOCK
5. iteration-2 测试验证质量约 T:6.5
