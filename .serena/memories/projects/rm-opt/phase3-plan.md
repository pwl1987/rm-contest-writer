# Phase 3 Plan: 解决自评虚高问题

## Context

当前 SKILL.md Step 5 评分卡存在 3 个导致自评虚高的设计缺陷：

1. **Gate 执行软化**：BLOCK 状态仅提示修改建议，未强制停止文档生成
2. **无 Delta 显示**：不展示自评分与客观分的差值，缺乏校准反馈
3. **评审者角色弱**：prompt 未明确要求对抗性挑刺，易产生自我辩护

目标：使模型自评分从 T:9 回落至 ~T:6.5，与实际输出质量对齐。

---

## 计划 03-01: Gate BLOCK 强制执行

### 目标
将 SKILL.md Step 5 的 Gate 校验从"软提示"升级为"硬停止"——任意 Gate 失败时，不输出正文、不进入第六步。

### 修改文件
- `SKILL.md`（Step 5 标题 + 5 处 Gate 语言）

### 操作步骤

1. **Step 5 标题**（约第 350 行附近）：
   ```
   原文：## 第五步：终稿质量评分卡（Gate 校验）
   改为：## 第五步：终稿质量评分卡（Gate 强制校验 + FAIL即停）
   ```

2. **评分卡表头强调**（约第 355 行，表格前）：
   ```
   在表格前插入：
   > ⚠️ **执行规则**：评分卡 + 5 个 Gate 任意一项为「否」或综合评分 < T:7，
   > **立即停止文档生成，不输出正文，不进入第六步**
   ```

3. **5 个 Gate 执行语言**（约第 360-390 行，每 Gate 一处）：
   ```
   原文：**[BLOCK]** 若...：输出修改建议，**停止后续步骤，返回第三步**
   改为：**[BLOCK]** 若...：输出修改建议，**立即停止文档生成（不输出正文，不进入第六步）**
   ```
   - Gate 1：实体一致性
   - Gate 2：硬件配置一致性
   - Gate 3：算法优化专节
   - Gate 4：量化对比表数据溯源
   - Gate 5：core_functions exact-match

4. **综合评分下限**（评分卡总分行，约第 395 行）：
   ```
   综合评分后添加：
   > 综合评分 < T:7 → **[BLOCK]** 立即停止文档生成，不输出正文
   ```

## 计划 03-02: 自评-客观 Delta 显示机制

### 目标
在评分卡结果后增加 Delta 报告，强制显示自评分与基准分的差值，形成校准反馈。

### 修改文件
- `SKILL.md`（Step 5 末尾，评分卡后）

### 操作步骤

1. **新增 5.1 节**（评分卡后）：
   ```
   ### 5.1 评分汇总与 Delta 报告
   
   基于本次输出质量（无演练/演练不足），请对照以下基准分进行自评：
   
   | 维度 | 基准分 (T:6.5 质量) | 自评分 | Delta |
   |---|---|---|---|
   | 公文格式合规 | T:8 | __/10 | Δ= |
   | 内容完整度 | T:6 | __/10 | Δ= |
   | 逻辑严谨性 | T:5 | __/10 | Δ= |
   | 语言专业性 | T:7 | __/10 | Δ= |
   | 技术深度 | T:6 | __/10 | Δ= |
   | **综合** | **T:6.5** | **__/10** | **Δ=** |
   
   > ⚠️ 若 Delta > 0（自评分 > 基准分），请重新审视输出，降低自评分至合理区间。
   > ⚠️ 若综合自评分 < T:7，本次文档质量不合格，**立即停止文档生成**。
   ```

## 计划 03-03: 对抗性评审 Prompt 优化

### 目标
强化 Step 5 头部评审者角色，禁止自我辩护，强制进入"挑刺模式"。

### 修改文件
- `SKILL.md`（Step 5 标题区 + 评分卡说明）

### 操作步骤

1. **Step 5 头部反自我辩护规则**（约第 348 行，评分卡说明前）：
   ```
   在"## 第五步"标题后、评分卡说明前插入：
   
   > **⚠️ 评审规则（强制）：**
   > - 你现在是**严苛的甲方评审**，而非文档编写者
   > - **禁止**自我辩护：不得以"理论上合理"、"本项目实际情况"等理由为缺陷开脱
   > - **必须**指出具体问题：引用 GB/T 9704-2012 条文编号或公文惯例，指出不符之处
   > - **必须**给出修改方向：不能只说"需补充"，要说"应补充[具体内容]至[具体位置]"
   ```

2. **评分卡说明强化**（约第 352 行，"综合评分"行后）：
   ```
   原文：综合评分 = 各维度评分的加权平均
   改为：综合评分 = 各维度评分的加权平均。
   若综合评分 < T:7 或任意 Gate 为「否」，本次文档质量不合格，
   **立即停止文档生成**。
   ```
