## 工作流程

### 第一步：动态信息采集（激活后立即执行）

**不询问用户，不让用户跳过。** 详细流程见 `references/采集问卷模板.md`

1. **检测历史项目** → 读取 `memory/index.json`，判断继续/新建
2. **关键词推断** → 赛道、AI部署、归属、核心功能（置信度≥70%直接采用）
3. **动态表单确认** → 展示推断结果，用户修改联动更新
4. **级联更新** → track变更联动架构图模板，描述信号变更联动Mockup
5. **采集完成** → 保存 `memory/projects/{project_id}.json`，进入第二步

---

### 第二步：方案评分（承上启下）

推荐 T:8+ 方案，说明 delta。展示核心功能关键词确认：

> **核心功能关键词确认**：方案中的核心功能必须与 `memory/projects/{project_id}.json` 中的 `core_functions` 字段精确匹配。

---

### 第三步：素材分析与结构搭建

基于采集信息，匹配赛道模板。**强制读取项目配置**：`model_choice`、`side`、`ai_mode` 必须从项目配置中读取并原样使用。

---

### 第四步：撰写成稿

严格按赛道模板（见 `references/赛道模板.md`）逐章撰写，关键数据插入表格对比。

---

### 第五步：终稿质量评分卡（Gate 强制校验）

执行规则见 `references/质量评分卡.md`。**5个[BLOCK] Gate任一触发 → 立即停止文档生成**，输出修改建议并返回第三步重新撰写。

**Gate 摘要：**
| Gate | 检查项 | 触发条件 |
|------|--------|---------|
| Gate 1 | 实体一致性 | model_choice、GPU型号、软件授权前后一致 |
| Gate 2 | 硬件配置一致性 | 仅技术研发类，GPU数量/型号/显存前后一致 |
| Gate 3 | 算法优化专节 | 仅技术研发类，覆盖≥2项算法优化手段 |
| Gate 4 | 量化对比表溯源 | 数据来源列存在，召回率行存在，效果数据为绝对值 |
| Gate 5 | core_functions exact-match | 3.2节必须完整出现所有core_functions词组 |

**综合评分 < T:7 → [BLOCK] 立即停止**

---

### 第六步：去冗余优化（强制执行）

1. 删除万能过渡句（"首先"、"其次"、"综上所述"）
2. 删除加强词（"非常"、"极其"、"十分"）
3. 合并重复表述
4. 被动句转主动句
5. **字数控制**：正文≤2500字，每段≤3行，功能描述≤50字/项
6. 效果数据全部使用绝对数值
7. 对比表5行自检（准确率/召回率/误报率/响应时延/并发量）

---

### 第七步：Mockup截图生成与嵌入

详细规则见 `references/Mockup匹配规则.md`

- **界面Mockup**：根据功能描述关键词匹配（与赛道无关）
- **架构图**：根据赛道选择模板
- **截图命令**：`python3 word-gen/capture-diagram.py <html路径> <输出png路径>`

## 大赛评审标准速查

### 通用标准（40 分，所有赛道适用）

| 得分项 | 分值 | 评审要点 |
|--------|------|---------|
| 原创性 | 10 | 创新点突出，文末附原创承诺 |
| 合规性 | 10 | 符合法规、舆论导向，内容积极健康 |
| 完整性 | 10 | 材料齐全，方案阐述清晰，核心内容完整呈现 |
| 实用性 | 10 | 贴合融媒实际需求，可落地可推广 |

### 赛道专项标准（60 分）

**技术研发类**（新系统/新算法/新工具开发）：
- 技术先进性 15（方案新颖 + 算法优化 + AIGC/Agent 突破）
- 应用效果 15（流畅 + 提效 + 体验佳）
- 实际应用价值 30（精准匹配痛点 + 显著提效 + 落地潜力极大）

**应用创新类**（业务流程优化类）：
- 应用创新性 15（创新优化 + 场景贴合 + 巧思）
- 落地可行性 30（逻辑严谨 + 步骤清晰 + 配套明确 + 适配实际）
- 数据有效性 15（数据完整 + 真实可溯源 + 印证价值）

**内容创意类**（视频/H5/海报/数据新闻等）：
- 创新性 15（新颖 + 融媒思维 + 记忆点）
- 可行性 30（受众 + 传播 + 落地路径 + 预期效益）
- 商业价值 15（产业引领 + 推广价值）

## 公文写作规范

- **禁用口语化表达**："蛮好的"、"挺不错" → 改为"效果良好"、"性能优异"
- **禁用加强词**："非常"、"极其"、"十分" → 直接删减或改为具体描述
- **禁用万能过渡**： "首先""其次""最后""综上所述" → 删除或用逻辑连接词替代
- **禁用模糊表述**："效果不错"、"提升明显" → 改为具体数值或量化描述
- **主语明确**：省略主语时需确认上下文清晰
- **主动优先**：减少被动句使用，主动句更直接有力
