# 动态信息采集详细流程

激活技能包后，立即启动**动态信息采集**，不询问用户，不让用户跳过。

---

## 1.0 检测历史项目

读取 `memory/index.json`：
- **有历史项目**：展示"继续 [项目名] 还是新建？"选项，用户确认后才继续
- **无历史项目**：自动进入动态分析模式
- **用户明确指定项目名**：直接加载该项目并进入撰写流程，跳过采集向导

---

## 1.1 关键词推断引擎

当用户提供项目描述后，AI 自动分析关键词，推断以下字段：

| 推断字段 | 关键词信号 | 置信度阈值 |
|---------|-----------|-----------|
| track（赛道） | "系统"→技术研发类 / "流程优化"→应用创新类 / "视频/H5/数据新闻"→内容创意类 | ≥70% 时直接采用，<70% 时列出选项让用户确认 |
| ai_mode（AI部署） | "多模态大模型"→私有化 / "API调用"→API模式 | 同上 |
| side（项目归属） | "选题"/"编辑"→日报侧 / "播出"/"节目"→广电侧 | 同上 |
| core_functions（核心功能） | 根据项目名称推断（见下表） | 同上 |

**核心功能推断表：**

| 项目关键词 | 推断功能类型 | 描述信号（匹配优先级） | 对应 Mockup 模板 |
|-----------|-------------|----------------------|-----------------|
| 选题/推荐/智能 | AI 推荐系统 | "推荐"、"发现"、"热点"、"选题" | dashboard + discovery + recommendation + trending |
| 审核/风控/合规 | 审核流程系统 | "审核"、"风控"、"合规"、"多级" | audit-workbench + process-config |
| 数据新闻/可视化/大屏 | 数据可视化 | "大屏"、"指挥"、"数据新闻"、"实时数据" | data-visualization + content-display |
| 视频/H5/海报 | 内容创意 | "视频"、"H5"、"海报"、"可视化" | content-display |
| 跨系统/互通/融合 | 平台融合 | "跨系统"、"互通"、"融合"、"集成" | dashboard + discovery |
| 监播/监控/信号/报警/切换/延时 | 广播监控系统 | "监播"、"信号"、"报警"、"延时" | dashboard + trending + preview + signal-monitor |

> **描述匹配规则**：根据用户提供的**功能描述**，而非赛道类型，自动匹配最接近的 Mockup 模板。同一项目可匹配多个模板（如"数据大屏"→ data-visualization + dashboard）。赛道只影响架构图模板，不影响界面 Mockup。

## 1.3 级联更新机制

当用户修改关键字段时，自动联动更新相关字段：

| 字段变更 | 联动更新 |
|---------|---------|
| track 变更 | key_tech_options / comparison_table_dims / resource_needs / **架构图模板（仅此项随赛道变化）** |
| ai_mode 变更 | model_options / hardware_needs |
| side 变更 | pain_points_options / existing_system_context |
| 描述信号变更 | **mockup_templates（界面 Mockup 根据功能描述匹配，与赛道无关）** |

**级联更新展示**：只列出受影响的字段变更，其他字段保持不变。

---

## 1.4 用户中途修改字段

用户说"把赛道改为应用创新类"，系统：
1. 检测字段变更
2. 执行级联更新（key_tech / comparison_table / resource_needs / mockup_templates 自动联动）
3. 展示变更记录供用户确认
4. 同步更新内存文件

---

## 1.5 采集完成

采集完成后：
1. 保存项目到 `memory/projects/{project_id}.json`
2. 展示采集结果摘要 + 对应 Mockup 模板预览列表
3. 进入第二步：方案评分
