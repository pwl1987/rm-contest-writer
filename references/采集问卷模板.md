# 动态信息采集详细流程

激活技能包后，立即启动**动态信息采集**，不询问用户，不让用户跳过。

---

## 1.0 检测历史项目

读取 `memory/index.json`：
- **有历史项目**：展示"继续 [项目名] 还是新建？"选项，用户确认后才继续
- **无历史项目**：自动进入动态分析模式
- **用户明确指定项目名**：直接加载该项目并进入撰写流程，跳过采集向导

---

## 1.1 关键词推断引擎

当用户提供项目描述后，AI 自动分析关键词，推断以下字段：

| 推断字段 | 关键词信号 | 置信度阈值 |
|---------|-----------|-----------|
| track（赛道） | "系统"→技术研发类 / "流程优化"→应用创新类 / "视频/H5/数据新闻"→内容创意类 | ≥70% 直接采用并预选；<70% 展示选项让用户确认 |
| ai_mode（AI部署） | "多模态大模型"→私有化 / "API调用"→API模式 | 同上 |
| side（项目归属） | "选题"/"编辑"→日报侧 / "播出"/"节目"→广电侧 | 同上 |
| core_functions（核心功能） | 根据项目描述推断（见下方映射表） | 同上 |

**核心功能 → Mockup 自动映射**（采集时自动适配）：

| 核心功能关键词 | 生成的 Mockup 类型 |
|--------------|-------------------|
| 监播/监控/信号/报警/切换/延时 | signal-monitor + dashboard |
| 推荐/发现/热点/选题 | dashboard + trending |
| 审核/风控/合规 | audit-workbench |
| 大屏/指挥/数据新闻/可视化 | data-visualization |
| 跨平台/互通/融合 | dashboard + discovery |
| 内容发布/H5/视频 | content-display |
| 架构图 | architecture（始终生成） |

> **规则**：赛道变更 → 架构图模板联动更新；核心功能关键词变更 → 界面 Mockup 联动更新。

---

## 1.2 对话式信息确认

推断完成后，生成确认表单，每项提供**推荐选项（预选）**，用户可直接确认或自由输入。

```
根据您的描述，为您的参赛项目推断如下：

1. 赛道
   ○ 技术研发类（推荐）○ 应用创新类 ○ 内容创意类
   或输入其他：___

2. AI部署模式
   ○ 私有化部署（推荐）○ API调用模式
   或输入其他：___

3. 项目归属
   ○ 日报社侧（推荐）○ 广播电视台侧
   或输入其他：___

4. 作品名称（请输入完整名称）
   推荐：智能选题推荐系统
   输入：___

5. 核心功能关键词（多选，推荐2-3个）
   ☑ 智能推荐  ☑ 热点追踪  ☐ 信号监测  ☐ 审核风控
   ☐ 数据大屏  ☐ 内容发布  ☐ 跨平台聚合
   或输入其他：___

6. 核心痛点（一句话描述）
   推荐：编辑依赖人工经验，选题效率低
   输入：___

7. 实施周期
   ○ 1个月 ○ 2个月（推荐）○ 3个月 ○ 半年以上
   或输入：___

8. 预算范围
   ○ 5-10万 ○ 10-30万（推荐）○ 30-50万 ○ 50万以上
   或输入：___

如需修改任何项，请直接说，例如：
"把赛道改成应用创新类"
"核心功能加上审核"
"作品名称叫智能监播系统"
```

**预选规则**：
- 置信度 ≥70% → 该选项预选为推荐项
- 置信度 <70% → 无预选，标红提示用户确认
- 用户可直接回复选项编号、选项名称，或自由输入

---

## 1.3 级联更新机制

当用户修改关键字段时，自动联动更新并告知：

| 字段变更 | 联动更新 | 告知用户 |
|---------|---------|---------|
| track 变更 | 架构图模板联动更新 | "赛道已更新为应用创新类，架构图模板已切换" |
| ai_mode 变更 | model_choice / hardware_needs 联动 | "AI部署模式已切换，硬件需求已更新" |
| side 变更 | pain_points_options 联动 | "项目归属已切换，痛点选项已更新" |
| 核心功能关键词变更 | Mockup 类型联动更新 | "核心功能已更新，相关Mockup已重新生成" |

**级联更新展示**：只告知受影响的变更，其他字段保持不变。

---

## 1.4 用户中途修改

用户说"把赛道改为应用创新类"：
1. 检测字段变更
2. 执行级联更新（架构图模板自动联动）
3. 展示变更结果，询问是否继续
4. 同步更新 `memory/projects/{project_id}.json`

---

## 1.5 采集完成

采集完成后：
1. 保存项目到 `memory/projects/{project_id}.json`
2. 展示采集结果摘要 + 对应 Mockup 预览列表
3. 进入第二步：方案评分
